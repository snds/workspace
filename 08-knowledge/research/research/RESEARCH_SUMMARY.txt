KEY FINDINGS
============

PRODUCTION READINESS:
- WebGPU is production-ready since Three.js r171 (September 2025)
- Zero-configuration imports: import * as THREE from 'three/webgpu'
- Automatic WebGL 2 fallback for 5% of users without WebGPU support
- Safari 26 (September 2025) completed browser support coverage

PERFORMANCE GAINS:
- 2-10x performance improvements in draw-call-heavy scenes
- Up to 96 FPS vs 24 FPS with clustered lighting
- 1 million particles updating in <1ms with compute shaders
- Significant CPU overhead reduction for GPU-bound workloads

TSL ADVANTAGES:
- Write once, compile to GLSL (WebGL) and WGSL (WebGPU)
- JavaScript-based with IDE support and refactoring
- Automatic compiler optimization (dead code elimination, uniform reuse)
- Type-safe node graphs
- Integrates with Three.js material lighting system

BROWSER COVERAGE (MARCH 2026):
- Chrome 113+ : Enabled (April 2023)
- Edge 113+   : Enabled (April 2023)
- Firefox 141+ (Windows), 145+ (macOS) : Enabled (July/Sept 2025)
- Safari 26+  : Enabled (September 2025)
- Coverage: ~95% of global users
- Fallback: WebGL 2 for remaining 5%

BREAKING CHANGES:
- ShaderMaterial / RawShaderMaterial → migrate to TSL + Node Materials
- EffectComposer → new node-based post-processing system
- Async initialization required (await renderer.init())

COMPUTE SHADER USE CASES:
- Particle physics updates (orders of magnitude faster)
- Physics simulations and constraints
- GPU-driven rendering (frustum culling, LOD selection)
- Image processing and filters
- Spatial data structure building (BVH, grid)
- Light culling for clustered shading

WEBGPU-SPECIFIC VISUAL TECHNIQUES:
- GPU-driven rendering with indirect draw calls
- Clustered lighting for 100+ lights without performance loss
- Bindless textures for material variety
- GPU culling and LOD selection
- Compute shader-based particle systems

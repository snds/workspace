# Game Systems Design for Legion

Systems are the core of Legion's gameplay. This reference covers economy, progression, production chains, and balance frameworks specific to Legion's hard sci-fi factory management × 4X × RTS structure.

## Economy Design

**Resource Types**
- **Matter** (elemental): Iron, Copper, Thorium, Rare Earth. Mined from asteroids/planets. Primary production input.
- **Energy** (power): Generated by reactors, stored in batteries. Gate all production. Scale with distance and infrastructure.
- **Data** (knowledge): Obtained by research, exploration, signals. Drives tech unlocks and clone learning.
- **Infrastructure Points** (IP): Built capacity—factory modules, weapons, sensors. Limited by surface area and power budget.

**Faucets and Sinks**

Faucets (sources of resources):
- Mining sites yield Matter at rate R_mine (affected by infrastructure, tech, distance)
- Reactors generate Energy at rate R_energy (affected by fuel type, research, placement)
- Research stations yield Data at rate R_data (affected by clone count, specialization, exploration)

Sinks (drains):
- Factory modules consume Matter (smelting, refining, construction)
- All production drains Energy (conversion loss, transmission loss)
- Combat consumes both Matter (ammo, repairs) and Energy (weapons, shields)
- Expansion costs Matter (infrastructure) and Data (unlocks)

**Inflation Control**

Resource overflow is the natural outcome of efficient mining. Three control mechanisms:

1. **Research Gates**: Major tech unlocks (better weapons, new factory types) require Data thresholds. Player can't skip tiers. Paces expansion.

2. **Dimensional Scaling**: As player owns more star systems, Energy transmission cost scales. Single-system optimization is cheap. Multi-system supply chains incur overhead.

3. **Threat Escalation**: Accumulated Matter triggers alien response events. Aggressive mining invites danger. Forces strategic restraint.

Target: Player should feel pressure to convert resources into progress (research, expansion) not accumulation (hoarding).

## Progression Systems

**Tech Tree Structure**

Three pillars:
- **Foundation**: Mining efficiency, basic smelting, reactor tech, habitat stability. Gated by early exploration.
- **Specialization**: Players choose one of three branches (Miner, Architect, Diplomat). Each unlocks unique factory types, units, and clone abilities.
- **Integration**: Late-game techs combine branches. Multi-system supply chains, merged research, faction diplomacy.

**Unlock Gates**

Gate progression by:
- **Data cost**: Higher tier = more Data required
- **Prerequisite tree**: Can't skip branches. Foundation → Specialization → Integration
- **Exploration gates**: Some techs require signals from alien relics, derelict ships, or specific star systems
- **Player choice gates**: Some branches mutually exclude (Miner favors strip-mining; Architect favors compact efficiency)

**Power Curves**

- **Early game** (first 5 hours): Single-star system. Factory grows 1.5× every resource tech unlock. Player feels rapid progression.
- **Mid game** (5-20 hours): Two-star system. Growth flattens. Energy transmission overhead introduced. Specialization unlocks create branching paths.
- **Late game** (20+ hours): Three-star system. Exponential growth returns via merged research and diplomacy. Mastery of combined systems enables rapid scaling.

Avoid straight exponential curves. Plateaus create decision points ("Expand now or research?").

## Factory / Production Chain Design

**Module System**

Modules are single-purpose buildings placed on a grid. Each module has:
- Footprint: 2×2, 3×3, etc.
- Input slot: accepts Matter type X
- Output slot: produces Matter type Y
- Energy draw: E_module
- Throughput: units/min
- Upgrade tiers: can improve throughput, reduce energy, or add secondary output

Example chain:
```
Iron Ore (mined) → Smelter (consumes 30/min, outputs Bar 20/min) → 
Assembly (Bar 20/min + Copper 10/min → Component 10/min)
```

**Throughput & Bottlenecks**

Modules should be balanced so:
- Input supply ≤ Processing throughput (mining output matches smelter input)
- Processing throughput ≤ Consumer throughput (factory output matches research/combat demand)

Create deliberate bottlenecks at choice points:
- A late-game rare material (Thorium) has limited sources. Player must specialize (Mine Thorium or buy from faction?).
- Assembly requires multiple input types. Player must manage logistics (parallel production lines or sequential batching?).

**Optimization Levers**

Give players tangible ways to optimize:
- Module placement: Closer modules reduce transmission loss (2% per unit distance)
- Specialization: Choosing Miner archetype increases mining speed +20%
- Tech unlock: Smelter tier 2 increases output 30%, reduces energy 10%
- Faction trade: Buy rare materials from NPC factions (costs Data, saves mining time)

Optimization should reward systems understanding, not just "bigger number go boom."

## Balance Frameworks

**Knob System**

Define adjustable parameters for each system:
- `mining_base_rate` = 20 Iron/min (all star systems)
- `smelter_throughput` = 15 Bar/min (can be upgraded to 20)
- `research_time_multiplier` = 1.0 (difficulty knob: 0.5 = faster, 2.0 = slower)
- `threat_escalation_threshold` = 500 (accumulated Matter before alien response)

Track all knobs in a central spreadsheet. When playtesting, adjust knobs, observe impact, iterate.

**Curves**

Define decision-point curves:
- Tech cost scales: Tier 1 = 100 Data, Tier 2 = 250 Data, Tier 3 = 500 Data (quadratic)
- Enemy difficulty scales: Star 1 = 1 enemy faction, Star 2 = 2 factions + anomalies, Star 3 = complex multi-faction conflict
- Factory area scales: Base zone = 30×30 grid, Upgrades +10×10 grid per tier (linear)

Test curves against playtime expectations. If mid-game research takes 3 hours for one unlock, players lose agency. Adjust curve.

**Playtesting Methodology**

1. **Run a session**: Player follows one specialization path. Log time to each milestone (first factory, first combat, first multi-system expansion).
2. **Collect metrics**: Mining time per resource type, research time distribution, combat win rate (should be ~70% with player skill).
3. **Identify bottlenecks**: Where does progression stall? Where does it accelerate too fast?
4. **Adjust one knob** (never multiple in one playtest). Retest.
5. **Document the change**: "Increased Thorium mining rate from 5/min to 8/min. Mid-game research now completes 25% faster. Reduced total playtime by ~1 hour."

## System Interconnections in Legion

The four systems form a feedback loop:

1. **Mining** → Matter flows into Factory
2. **Factory** → Produces research goods and military supplies
3. **Research** → Unlocks better Mining, Factory, and Combat tech
4. **Combat/Expansion** → Consumes Factory goods, creates new systems to Mine

A player's choice in *specialization* affects the loop:
- **Miner**: Prioritizes throughput. Scales horizontably (many small factories).
- **Architect**: Prioritizes efficiency. Scales vertically (few dense factories).
- **Diplomat**: Mixes both, trades with factions to break resource bottlenecks.

All three should be viable. None should dominate. Playtesting verifies this.
